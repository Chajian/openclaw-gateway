◆ 项目定位

OpenClaw 是一个开源 AI Agent 网关（MIT 协议，TypeScript）， 支持多渠道消息（Telegram / WhatsApp / Slack / Discord）、多模型路由、技能系统、定时任务。

本仓库 claw 围绕 OpenClaw 构建了一套自动化 API Key 供给链路： 从 LinuxDo 社区自动发现公益站 → OAuth 注册登录 → 同步 Key & 配额 → 挑选最优 Key 写入网关配置。

⚠ 踩坑指南

前人踩过的坑，后人不必重蹈覆辙

  • 1. 网关启动报 "Missing config"
    原因：~/.openclaw/openclaw.json 不存在或缺少 gateway.mode
    修复：创建配置，至少包含 {"gateway":{"mode":"local"}}
  • 2. "non-loopback Control UI requires allowedOrigins"
    原因：OPENCLAW_GATEWAY_BIND=lan 绑定到 LAN 时需要配置 CORS
    修复：添加 controlUi.allowedOrigins + dangerouslyAllowHostHeaderOriginFallback: true
  • 3. CLI 容器无法连接网关
    原因：docker compose run openclaw-cli 连接的是自己容器的 localhost
    修复：始终用 docker compose exec openclaw-gateway
  • 4. Telegram "chat not found"
    原因：用户未给 Bot 发送过消息
    修复：先在 Telegram 给 Bot 发 /start
  • 5. Telegram getUpdates 409 冲突
    原因：网关在长轮询消费 updates，外部调用冲突
    修复：不要同时调 getUpdates；获取 Chat ID 时先禁用 Telegram
  • 6. Docker build 拉不到镜像（EOF / DNS 失败）
    原因：Docker Hub 需要代理
    修复：~/.docker/daemon.json 配置 proxies，Desktop 设 manual 代理 host.docker.internal:7890
  • 7. LinuxDo RSS 返回 0 篇帖子
    原因：Cloudflare 429 限流
    修复：增大检查间隔（600s），请求间加 3s 延迟，使用浏览器 UA
  • 8. dmPolicy "open" 报错
    原因：dmPolicy: "open" 必须配合 allowFrom: ["*"]
    修复：同时设置 "allowFrom": ["*"]

🔒 安全须知

永远不要提交 data/ 目录和真实的 config/sites.json。API Key、密码、Token 只存在加密仓库中。
KEYHUB_MASTER_KEY 必须设置为环境变量，不要写入代码或提交到仓库。
部分站点强制 CAPTCHA/Turnstile/OAuth 手动授权，onboard:linuxdo 会标记为 manual_required，需手动处理后重跑。
